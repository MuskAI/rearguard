# 慧鉴 AI 多角色系统审查

审查日期：2026-07-19

## 审查视角

- 安全与隐私：租户隔离、会话边界、CSRF、代理头、媒体与报告授权
- SRE 与平台工程：部署原子性、回滚、健康检查、备份、任务恢复、模型常驻
- 法务与电子证据：原件哈希、首次固化、签名清单、结论一致性、删除与留存
- 商业化与计费：API Key、额度预占、幂等结算、失败释放、用量审计
- 产品与交互：检测模式、失败恢复、账户切换、分享、可访问性和反馈闭环
- 模型服务：GPU 并发、排队、超大图下采样、切块上限、水印并行检测

## 已关闭的高风险问题

### 账号与数据隔离

- 正式账户统一使用不可变 `account_uuid`，不再按手机号、openid 或两个数据库各自的自增 ID 猜测归属。
- 图像、视频、开发者任务、幂等键、历史详情、媒体和报告都执行 UUID 精确匹配。
- Web 与内部检测进程透传同一个 `account_uuid`，首条检测记录即完成归属，不再为已登录用户额外产生未归属副本。
- 未绑定 UUID 的旧记录默认不可见，必须经管理员核实后离线修复。
- 前端账户切换时清空私有状态，并忽略上一账号迟到的异步响应。

### 证据与报告

- V1 图像报告首次生成时固化独立服务端证据快照，并签入原件 SHA-256、大小、模型版本、策略版本、结论、水印和实拍元数据摘要。
- 快照固化失败时检测失败关闭；Swarm 只在最终聚合结论形成后固化，避免签入中间结论。
- 边界分数与低置信度统一签为“需人工复核”，同时保留原始模型标签，避免报告正文与签名清单冲突。
- 用户删除已归属历史时同步删除证据快照；损坏的普通快照仍可删除，符号链接或硬链接异常会被拒绝。
- PDF 文案明确这是完整性校验与辅助判断，不使用“权威证据”或“司法结论”等越界表述。

### 开发者平台与计费

- API 检测任务改为私有 spool、独立 worker、数据库租约、心跳、幂等键和重启恢复。
- 额度预占、任务结算、用量事件和账本在同一事务中完成；每个任务只允许一个用量事件。
- 失败任务释放预占额度，任务保存 token 用量，恢复结算不会丢失统计。
- 缺少明确 `verdict` 授权的旧成功任务按 `review_only` 处理；已经错误结算的任务会在行锁事务中幂等退回免费次数或余额、释放日额度、写入冲正账本，并把用量修正为非计费。
- Web 最终响应日志与开发者任务共用失败关闭的结果契约；旧日志缺字段时只恢复为待人工复核，不能恢复为带概率的自动结论。
- API Key 轮换在事务与行锁内完成，避免并发产生多个有效替换 Key。
- API Key 创建/轮换的幂等派生使用独立、持久的专用密钥，不再随内部服务鉴权密钥轮换而改变。
- 新 API Key 默认只授予快速图像检测，Swarm 与报告下载必须由用户显式勾选；旧 `detect` scope 不再隐式获得 Swarm 或报告权限。
- 结算用量事件由数据库唯一键和 `INSERT IGNORE` 双重约束；复核态退回计费额度但保留每日请求计数，避免通过失败任务绕过滥用限额。

### 稳定性与性能

- V1 readiness 检查账户库、检测库、队列最老年龄、worker 维护状态与任务 claim 健康状态；进程存活但失去数据库写权限时失败关闭。V2 readiness 对 SQLite 执行可回滚写探针。
- 网页快速检测、Swarm 和开发者 API 共用数据库租约 worker，但使用独立容量池、按账号/访客限额，并为 Web 与开发者 API 各保留一个执行槽位。
- Web 任务以 `job_id` 关联业务历史并记录 effect/result；只有完整最终响应日志存在时才能恢复成功。仅有原始模型行的半成品任务失败关闭，不会发布为完整结论或自动重复推理。
- 访客免费检测使用服务端来源散列做每日限额，IPv6 按 `/64` 聚合并设置全站每日预算；清空 Cookie 或轮换 IPv6 地址不能重置额度，高成本 Swarm 仅对登录用户开放。
- 图像推理在解码和切块前获取 GPU 槽位，限制源图像素与最多切块数，超大图先下采样。
- GPU 服务保持一个常驻模型进程，使用线程并发，不再按请求数周期重启造成模型冷加载和显存重叠。
- 主模型原始 softmax 分数只有在独立 Ed25519 签名校准清单通过后才允许形成自动结论；清单绑定 ONNX、推理实现、决策门禁实现、完整运行时锁、预处理、类别映射、切块/Top-K 参数、数据集、评测代码、阈值、FPR/FNR 和有效期。模型服务器与公网 Web 均只持有同一只读公钥，Web 和报告生成器会再次验签。
- V2 语义视觉模型同样执行显式 `decisionStatus`/`decisionAuthority` 门禁；模型响应不能自行声明决定权。未校准模型、旧缓存和缺失授权字段的历史记录统一降级为 `review_only`，公开风险向量为空。
- 水印坐标不再被截断或“修正”；负数、NaN、Infinity 和越界框直接拒绝。YOLO 离线时不能声称“未检出”；注册表命中的平台标记也只作为视觉归属线索，不单独形成真假结论。
- 每次 GPU 推理的原始分数、发布分数、原图/处理后尺寸、下采样信息、切块数量、参数和运行时身份均进入服务端审计，并随首次证据快照签名。
- 缺少完整推理审计的校准模型结果同样降级；`review_only` 页面、后台统计、HTML/PDF 报告均不再把 `50%` 当作风险占位值。
- 取证与内容凭证接口增加并发信号量和排队超时，避免请求堆积拖垮事件循环。
- 前端等待超时后保留原任务 ID 并继续查询，不会提示用户重复上传、重复扣额。
- V2 分析缓存按 `account_uuid` 或 API Key 租户散列隔离，匿名/内部请求不复用分析缓存；相同文件不能跨账号命中私有分析结果。
- 登录会话增加 7 天绝对上限，滑动续期不能无限延长；退出登录仅接受 POST，避免跨站 GET 触发状态变更。

### 发布与恢复

- 部署脚本拒绝发布路径中的未提交改动，`DEPLOYED_COMMIT` 与实际制品保持一致。
- V1/V2 后端采用版本目录和原子指针切换；前端保留上一版本至全部健康检查通过。
- 激活失败会恢复上一版后端、前端和 Nginx 配置并重启服务。
- V2 依赖由 `uv.lock` 导出的哈希锁定文件安装，systemd 单元纳入版本控制并限制写目录。
- V1 新 release 与独立虚拟环境在旧服务在线时预构建，停机窗口只用于迁移、指针切换和重启。
- 66 与公网服务器在切换前分别安装 15 分钟 TTL 回滚任务和独立发布锁；即使发布机断网或被强制终止，两端也会独立收敛。主模型、水印、YOLO、完整运行时快照、隧道、公网 detector 配置和 systemd 单元作为同一版本切换。watchdog 不在发布机内顺序取消，而是在到期时核验双端 commit、CUDA/水印服务和公网 worker 状态：成功版本自动 no-op，半发布版本恢复文件及服务原有 active/enabled/masked 状态，过期事务不能覆盖后续版本。
- 每次迁移前创建并校验备份；网页任务和开发者任务均由持久队列恢复或按幂等业务结果收口。
- GPU 响应签名携带 key ID，Web 端支持历史验证 keyring；GPU 可临时接受前一版内部 token。部署器核对两端活动密钥指纹和 key ID，使两类密钥都可按先扩散验证能力、后切换签名方的顺序零停机轮换。

- 公网访客快速检测完成时间约 8 秒，GPU 推理、原件归档和首次签名证据快照均成功。
- 同一访客读取原图和 PDF 为 `200`；另一访客读取该原图为 `401`。
- 临时开发者 API 账号完成一次快速检测约 9 秒：任务从 queued/running 进入 success，免费额度预占归零、已用次数加一、用量事件和账本各一条。
- 开发者 API 原图与 PDF 均为 `200`，PDF 头为 `%PDF`；检测库只产生一条带 UUID 的记录，未归属历史数量没有增加。
- 66 算法服务器主进程无重启，模型常驻 CUDA；健康信息显示 `CUDAExecutionProvider`、并发上限 2、预热 26 视图。
- 公网 V1/V2 readiness、Nginx 和四个核心服务均通过。生产冒烟账号、Key、任务、计费、原件和证据快照已在验证后精准清理。

这些冒烟结果验证了当前单请求主链路，不替代容量测试、故障演练或长期稳定性观察。

## 仍未关闭的商业发布门槛

### P0：自动真假判定上线前必须完成

- 66 当前 ONNX 模型在 4 张已知实拍照片上输出 90.57% 至 99.99% 的 AI 原始分数，尚未达到自动判定的商用准确率要求。
- 使用独立真实相机、社交平台压缩和主流生成模型校准集核验标签顺序、FP32/INT8 对齐、预处理、分辨率分层、阈值、FPR/TPR；将通过评审的样本量、FPR/FNR、阈值和数据集 SHA 写入模型校准记录后，代码门禁才会开放自动结论。
- 在此之前系统保持 `review_only`：普通模型分数与可见平台标记只能触发人工复核；只有校验通过的内容凭证可形成独立来源结论。

### P1：正式商业发布前需要完成

- 对现存 699 条未绑定 `account_uuid` 的旧图像记录做人工归属映射或按留存策略归档；系统继续对它们失败关闭。
- 配置异地 `rclone` 目标，从远端副本再执行一次隔离恢复演练。当前本机快照恢复已通过，但仍不能覆盖整机或云盘故障。
- 代码已支持 GPU 响应密钥与证据签名历史 keyring；运维仍需建立正式轮换台账。商业取证场景应迁移到 KMS/HSM，并将快照副本写入 WORM 或等价不可变存储。
- 为生产告警接入外部通知渠道，并从公网节点验证 Web、V1 readiness、V2 readiness、66 模型隧道和备份新鲜度。

### P2：下一阶段架构升级

- 将视频报告与 V2 深度取证报告统一到图像报告的首次固化、签名、密钥轮换和验签流程。
- 将本机 spool 和 SQLite 迁移到共享对象存储与高可用数据库后，才能安全水平扩展到多 Web 节点。
- 增加历史记录游标分页、完整键盘焦点陷阱和屏幕阅读器回归测试。

## 发布判断

当前代码可作为受控、人工复核优先的试运行候选版本；未校准主模型不会再发布确定性真假结论。完成模型 P0 与上述 P1 运维门槛前，不应宣称“自动鉴伪达到商用准确率”“高可用”“司法取证级”或“无限并发”。

容量与延迟基线见 `docs/CAPACITY_TEST_REPORT_2026-07-19.md`。当前推荐总执行并发为 2，突发流量由持久队列吸收。
