# 慧鉴 AI 多角色系统审查

审查日期：2026-07-19

## 审查视角

- 安全与隐私：租户隔离、会话边界、CSRF、代理头、媒体与报告授权
- SRE 与平台工程：部署原子性、回滚、健康检查、备份、任务恢复、模型常驻
- 法务与电子证据：原件哈希、首次固化、签名清单、结论一致性、删除与留存
- 商业化与计费：API Key、额度预占、幂等结算、失败释放、用量审计
- 产品与交互：检测模式、失败恢复、账户切换、分享、可访问性和反馈闭环
- 模型服务：GPU 并发、排队、超大图下采样、切块上限、水印并行检测

## 已关闭的高风险问题

### 账号与数据隔离

- 正式账户统一使用不可变 `account_uuid`，不再按手机号、openid 或两个数据库各自的自增 ID 猜测归属。
- 图像、视频、开发者任务、幂等键、历史详情、媒体和报告都执行 UUID 精确匹配。
- 未绑定 UUID 的旧记录默认不可见，必须经管理员核实后离线修复。
- 前端账户切换时清空私有状态，并忽略上一账号迟到的异步响应。

### 证据与报告

- V1 图像报告首次生成时固化独立服务端证据快照，并签入原件 SHA-256、大小、模型版本、策略版本、结论、水印和实拍元数据摘要。
- 快照固化失败时检测失败关闭；Swarm 只在最终聚合结论形成后固化，避免签入中间结论。
- 边界分数与低置信度统一签为“需人工复核”，同时保留原始模型标签，避免报告正文与签名清单冲突。
- 用户删除已归属历史时同步删除证据快照；损坏的普通快照仍可删除，符号链接或硬链接异常会被拒绝。
- PDF 文案明确这是完整性校验与辅助判断，不使用“权威证据”或“司法结论”等越界表述。

### 开发者平台与计费

- API 检测任务改为私有 spool、独立 worker、数据库租约、心跳、幂等键和重启恢复。
- 额度预占、任务结算、用量事件和账本在同一事务中完成；每个任务只允许一个用量事件。
- 失败任务释放预占额度，任务保存 token 用量，恢复结算不会丢失统计。
- API Key 轮换在事务与行锁内完成，避免并发产生多个有效替换 Key。

### 稳定性与性能

- V1 readiness 检查账户库、检测库和开发者 worker 心跳；V2 readiness 对 SQLite 执行可回滚写探针。
- 图像推理在解码和切块前获取 GPU 槽位，限制源图像素与最多切块数，超大图先下采样。
- GPU 服务保持一个常驻模型进程，使用线程并发，不再按请求数周期重启造成模型冷加载和显存重叠。
- 取证与内容凭证接口增加并发信号量和排队超时，避免请求堆积拖垮事件循环。
- 前端等待超时后保留原任务 ID 并继续查询，不会提示用户重复上传、重复扣额。

### 发布与恢复

- 部署脚本拒绝发布路径中的未提交改动，`DEPLOYED_COMMIT` 与实际制品保持一致。
- V1/V2 后端采用版本目录和原子指针切换；前端保留上一版本至全部健康检查通过。
- 激活失败会恢复上一版后端、前端和 Nginx 配置并重启服务。
- V2 依赖由 `uv.lock` 导出的哈希锁定文件安装，systemd 单元纳入版本控制并限制写目录。
- 每次迁移前创建并校验备份；网页进程中断的任务会明确失败，不再永久停留在“处理中”。

## 仍未关闭的商业发布门槛

### P1：上线前需要完成

- 配置异地 `rclone` 备份并执行一次隔离环境恢复演练，记录实际 RPO/RTO。
- 为证据签名密钥建立正式轮换台账；商业取证场景应迁移到 KMS/HSM，并将快照副本写入 WORM 或等价不可变存储。
- 执行包含登录、快速检测、Swarm、报告和开发者 API 的容量压测，给出并发上限、P95/P99、错误率和限流阈值。
- 为生产告警接入外部通知渠道，并从公网节点验证 Web、V1 readiness、V2 readiness、66 模型隧道和备份新鲜度。

### P2：下一阶段架构升级

- 将网页快速检测和 Swarm 也迁移到持久任务队列。目前重启会正确标记失败，但不能自动续跑。
- 将视频报告与 V2 深度取证报告统一到图像报告的首次固化、签名、密钥轮换和验签流程。
- 将本机 spool 和 SQLite 迁移到共享对象存储与高可用数据库后，才能安全水平扩展到多 Web 节点。
- 增加历史记录游标分页、完整键盘焦点陷阱和屏幕阅读器回归测试。

## 发布判断

当前代码可作为受控商业试运行候选版本：代码可闭合的 P0 问题已经处理并有回归覆盖。完成上述 P1 运维门槛前，不应宣称“高可用”“司法取证级”或“无限并发”。
