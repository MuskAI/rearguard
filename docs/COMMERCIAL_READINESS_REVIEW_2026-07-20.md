# 慧鉴 AI 商用稳定性复审

日期：2026-07-20

## 结论

本轮对多租户、管理员权限、模型结论治理、API Key、审计、备份、恢复、告警、Nginx 和部署回滚进行了三轮跨行业复审。代码层未保留已知 P0；未经独立校准的主模型继续强制 `review_only`，因此当前系统可以安全提供“证据与人工复核”服务，但不能对外宣称自动真假判定已经达到商用准确率。

## 已完成

- V1/V2 历史、报告、媒体和删除接口按不可变账号 UUID 隔离；开发者 Key 读取报告必须具备 `reports` scope。
- 用户和管理员退出会递增服务端会话版本，复制的旧 Cookie 立即失效。
- VLM 取证只输出证据观察，顶层和自然语言中的真假结论、概率、置信度均被移除或隐藏。
- GPU 运行时在线与“自动结论已授权”分开显示；未通过独立校准门禁时只允许人工复核。
- 旧记录认领采用原媒体哈希、证据文件哈希、目标账号锁、双角色审批、同人员拦截、HMAC 封印、EXIF 冲突拒绝和可驳回释放。
- API Key 创建、轮换、吊销与 HMAC 审计事件在同一数据库事务中提交；每五分钟执行全链验证和单调文件检查点核对。
- 备份状态原子落盘；独立告警 worker、dead-man watchdog、每日备份和每周隔离恢复演练由 systemd 管理。
- 公网默认虚拟主机拒绝未知 Host，预览端口只监听回环地址，大屏和公开报告凭据不进入访问日志。

## 外部阻断项

以下事项无法仅靠仓库代码完成，未完成前不得标记“完全商用就绪”：

1. 使用独立、冻结、可追溯的真实/生成数据集完成 FP32/部署运行时对齐、FPR/FNR、阈值和漂移验收；当前已知实拍样本仍会被主模型高分误判。
2. 配置异地 `rclone`、KMS/HSM 和 WORM/对象锁存储，将安全审计检查点写入独立信任域并实际完成一次异地恢复演练。
3. 配置真实告警 Webhook 并完成断网、进程退出、备份失败和恢复失败演练。
4. 为 operator/reviewer 接入不可自声明的员工 IdP/MFA；当前只能以后台账号、角色、管理员 ID 和绑定手机号执行职责分离。
5. 若 SLA 要求容忍整机或机房故障，需要增加 Web、MySQL、GPU 推理和对象存储的冗余实例；当前仍有单机故障域。
